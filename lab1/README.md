# Лабораторна робота: Автотестування та CI/CD Pipeline

Цей репозиторій містить виконання лабораторної роботи, метою якої є налаштування базового робочого процесу (Git Flow) з використанням автотестів та GitHub Actions. 

В якості тестового завдання реалізовано логіку розрахунку індексу маси тіла (ІМТ) з вбудованими перевірками на некоректні дані (щоб уникнути ділення на нуль та інших небажаних операцій).

## Виконані критерії здачі

- [x] **Код:** Написано базовий функціонал мовою Python (не калькулятор).
- [x] **Автотести:** Написано 5 автотестів з використанням `pytest` (включаючи позитивні сценарії та перевірки на виключення `ValueError`).
- [x] **Успішність:** Усі автотести проходять успішно та не падають.
- [x] **CI/CD Pipeline:** Налаштовано GitHub Actions Workflow, який запускається автоматично при створенні Pull Request у головну гілку.
- [x] **Кроки пайплайну:** Пайплайн включає перевірку стилю коду (linting), запуск тестів та збереження HTML-звіту про тестування як артефакту (export results).

## Структура проєкту

* `.github/workflows/pr-pipeline.yml` — конфігурація пайплайну GitHub Actions.
* `fitness.py` — основний файл з бізнес-логікою (функції розрахунку).
* `test_fitness.py` — файл з тестами для перевірки логіки.
* `requirements.txt` — список залежностей проєкту.

## Локальний запуск тестів

Щоб запустити проєкт та тести локально на вашому комп'ютері, виконайте наступні команди:

1. Встановіть необхідні залежності:
   ```bash
   pip install -r requirements.txt
2. Запустіть тести:
   pytest
3. Для генерації HTML-звіту (опціонально):
   pytest --html=test-report.html --self-contained-html
Робочий процес (Pull Request)

Всі нові зміни розробляються в окремих гілках (наприклад, feature/fitness-logic) і зливаються в main виключно через механізм Pull Request. Це гарантує, що неперевірений код не потрапить у головну гілку до успішного проходження всіх тестів у GitHub Actions.
