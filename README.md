# Postmodern-Technologies-in-Software-Development

Цей репозиторій містить практичні матеріали та виконані лабораторні роботи з курсу. Головна мета проєкту — побудова сучасного процесу розробки, тестування та розгортання програмного забезпечення (CI/CD).

## 🛠 Технологічний стек
* **Мова програмування:** Python 3.10+
* **Тестування:** Pytest, Flake8 (Linting)
* **Контейнеризація:** Docker
* **CI/CD:** GitHub Actions, GitHub Container Registry (GHCR)
* **Оркестрація (додатково):** Kubernetes (Minikube), Helm

---

## 📦 Лабораторна робота 1: Git Flow, Автотести та CI для Pull Requests

**Що реалізовано:**
* Написано скрипт на Python із бізнес-логікою розрахунку індексу маси тіла (ІМТ) та перевірками на некоректні вхідні дані (захист від ділення на нуль, від'ємних значень тощо).
* Створено набір з 5+ автотестів за допомогою фреймворку `pytest` (включаючи позитивні та негативні сценарії).
* Налаштовано безпечну роботу з репозиторієм через SSH-ключі.
* **CI Pipeline:** Створено GitHub Actions Workflow (`pr-pipeline.yml`), який автоматично запускається при створенні Pull Request у головну гілку. Пайплайн виконує:
  1. Перевірку стилю коду (Linting).
  2. Запуск автотестів.
  3. Генерацію та збереження HTML-звіту про результати тестування як артефакту (Export results).

---

## 🐳 Лабораторна робота 2: Docker, CI/CD та Розгортання

**Що реалізовано:**
* Проєкт запаковано у Docker-контейнер (написано `Dockerfile` та оптимізовано збірку).
* **CI/CD Pipeline:** Створено Workflow (`docker-ci.yml`), який реагує на пуш у гілку. Пайплайн:
  1. Збирає Docker-образ.
  2. Автоматично форматує назву образу згідно з вимогами Docker (видалення спецсимволів).
  3. Публікує готовий образ у GitHub Container Registry (GHCR).
  4. Завантажує образ і запускає всередині нього тести для фінальної перевірки.
* **Додаткове завдання:** Згенеровано Helm Chart (`lab2-chart`) для розгортання зібраного Docker-образу в локальному кластері Kubernetes (Minikube).
